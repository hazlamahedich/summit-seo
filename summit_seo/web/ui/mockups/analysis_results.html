Critical
The checkout page includes HTTP resources on an HTTPS page, creating mixed content. This can lead to security warnings, browser blocking, and reduced user trust.
<script src="http://example-analytics.com/tracker.js"></script>
<img src="http://cdn.example-store.com/images/product-123.jpg" alt="Product Image">
How to Fix
Update all resource URLs to use HTTPS instead of HTTP to ensure secure connections.
  • Replace all HTTP references with HTTPS in your HTML, CSS, and JavaScript files
  • Ensure that all external resources (scripts, images, iframes) support HTTPS
  • Implement Content-Security-Policy headers to prevent mixed content
